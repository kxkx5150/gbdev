GB ASMチュートリアル

入力

ゲームとしての基礎はできているのですが、プレイヤーの意見が不足しています。 勝手にプレイするゲームでは面白くないので、そこを直していきましょう。

このコードをMainループの下に貼り付けます。Memcpyと同様に、call命令を使って、別の場所から再利用できる関数です。

Input: ; コントローラの半分をポーリング ld a,P1F_GET_BTN call.onenibble ld b, a ; B7-4 = 1; B3-0 = 押されていないボタン
 ; 残りの半分をポーリング ld a,P1F_GET_DPAD call.onenibble swap a ; A3-0 = 押されていない方向 ; A7-4 = 1 xor a, b ; A = 押されたボタン + 方向 ld b, a ; B = 押されたボタン + 方向
 ; そしてコントローラを解放 ld a,P1F_GET_NONE ldh [rP1], a
 ;前のwCurKeysと組み合わせてwNewKeysに ld a, [wCurKeys] xor a, b ; A = 状態が変化した キーと a, b ; A = 押されたキーに変化 ld [wNewKeys], a ld a, b ld [wCurKeys], a ret
.onenibble ldh [rP1], a ;キーマトリックスを切り替える call.knownret;既知のretを呼び出す10サイクルを燃やす ldh a, [rP1] ; キーマトリックスが解決するのを待つ間値を無視 ldh a, [rP1] ldh a, [rP1] ; この読み込みは数える または a, $F0 ; A7-4 = 1; A3-0 = 非押しキー.knownret ret

残念ながら、ゲームボーイの入力の読み取りはかなり複雑で（見ての通り！）、今この関数が何をしているのかを説明するのはかなり難しい。 だから、例外的に、この関数が入力を読み取っていることを信じて欲しい。 いいな？よしっ

関数の使い方がわかったところで、メインループの中でUpdateKeys関数を呼び出して、ユーザーの入力を読み込んでみましょう。UpdateKeysは、押したボタンをwCurKeys と呼ぶメモリ上の場所に書き込み、関数が戻った後にそこから読み込むことができます。 このため、UpdateKeysはフレームごとに 1 回だけ呼び出せばよいのです。

これは重要なことです。すでに処理した入力を再ロードする方が速いだけでなく、プレイヤーがフレームの途中でボタンを押したり離したりしても、常に同じ入力で動作することになるからです。

まず、UpdateKeysが使用する2つの変数を入れるスペースを確保します。main.asmの最後に以下のように貼り付けてください。

SECTION "入力変数", WRAM0wCurKeys: dbwNewKeys: db

Each variable must reside in RAM, and not ROM, because ROM is “Read-Only” (so you can’t modify it). Additionally, each variable only needs to be one byte large, so we use db (“Define Byte”) to reserve one byte of RAM for each.

We’re going to use the and opcode, which we can use to set the zero flag (z) to the value of the bit. We can use this along with the PADF constants in hardware.inc to read a particular key.

Main:
    ld a, [rLY]
    cp 144
    jp nc, Main
WaitVBlank2:
    ld a, [rLY]
    cp 144
    jp c, WaitVBlank2

    ; Check the current keys every frame and move left or right.
    call Input

    ; First, check if the left button is pressed.
CheckLeft:
    ld a, [wCurKeys]
    and a, PADF_LEFT
    jp z, CheckRight
Left:
    ; Move the paddle one pixel to the left.
    ld a, [_OAMRAM + 1]
    dec a
    ; If we've already hit the edge of the playfield, don't move.
    cp a, 15
    jp z, Main
    ld [_OAMRAM + 1], a
    jp Main

; Then check the right button.
CheckRight:
    ld a, [wCurKeys]
    and a, PADF_RIGHT
    jp z, Main
Right:
    ; Move the paddle one pixel to the right.
    ld a, [_OAMRAM + 1]
    inc a
    ; If we've already hit the edge of the playfield, don't move.
    cp a, 105
    jp z, Main
    ld [_OAMRAM + 1], a
    jp Main

Now, if you compile the project, you should be able to move the paddle left and right using the d-pad!! Hooray, we have the beginnings of a game!
